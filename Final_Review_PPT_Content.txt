SLIDE 2 — Problem Statement
═══════════════════════════════════════════════════════════

- Traditional exams use fixed difficulty for every student.
- Marks alone fail to reflect actual competency level.
- No domain-wise analysis (Aptitude / Reasoning / Verbal).
- No tracking of learning progression over time.
- Static question banks expose questions through repeated practice.
- Need: an intelligent, adaptive, AI-driven competency analyser.


═══════════════════════════════════════════════════════════
SLIDE 3 — Objectives
═══════════════════════════════════════════════════════════

Main Objective:
- To design and implement a Multi-Agent System for adaptive,
  domain-wise student competency analysis using LLM-generated questions.

Specific Objectives:
- To develop a CrewAI-based three-agent pipeline mapping to the APOT model.
- To integrate Gemini 2.5 Flash for on-the-fly MCQ generation across three domains.
- To dynamically adjust question difficulty using correctness and response time.
- To compute a Competency Index quantifying per-domain and overall competency.
- To deliver a Django + MySQL web platform with a real-time analytics dashboard.

SLIDE 6 — Explanation of Components
═══════════════════════════════════════════════════════════

- User Registration & Login — Django auth; session-bound test attempt.
- Dashboard / Domain Selection — Pick Aptitude, Reasoning, or Verbal Ability.
- Question Generator — Gemini 2.5 Flash; live MCQ via JSON schema.
- MCQ Evaluator Agent (SoLMA – Assess) — Outputs correctness verdict + reason.
- Difficulty Predictor Agent (SoLPA – Predict) — Classifies Easy / Medium / Hard.
- Adaptive MCQ Overseer Agent (SoLTA – Oversee + Transit) — Sets next difficulty.
- Competency Index DB — Stores per-question result, time, difficulty.
- AI Feedback Module — Generates summary, leaderboard band, recommendations.


═══════════════════════════════════════════════════════════
SLIDE 7 — Algorithms / Models Used
═══════════════════════════════════════════════════════════

- APOT Model (Outay et al., 2024) — four-phase adaptive learning loop.
- CrewAI Sequential Process — three-agent orchestration with shared context.
- Gemini 2.5 Flash (LLM) — live MCQ generation under JSON schema.
- Psychometric Rule Classifier — Fast+Correct → Easy / Slow+Correct → Medium / Wrong → Hard.
- Vygotsky's Zone of Proximal Development + Bloom's Taxonomy — drive next-question logic.
- Mathematical Indices — Learning Index (LI), Stage of Learning Index (SoLI), Competency Index (CI).

Novelty:
- First implementation that fuses CrewAI + LLM-generated MCQs onto the APOT cycle, replacing static item banks.

SLIDE 9 — Dataset Description
═══════════════════════════════════════════════════════════

- Source: Live LLM generation — Gemini 2.5 Flash (no fixed item bank).
- Domains (3): Aptitude, Reasoning, Verbal Ability.
- Difficulty levels (3): Easy, Medium, Hard.
- Session size: 20 MCQs per attempt, with response time per question.
- Per-question fields captured: question, options, correct answer,
  student answer, time(s), domain, difficulty.
- Persistence: MySQL — StudentTestResult, QuestionPerformance, AdaptiveLearningHistory.
- Train/Val/Test split: Not applicable — no model training; system is rule + LLM driven.
